Timer code compiliation
by Andrew Whittle - Tuesday, 24 March 2015, 5:09 PM
 

So when trying to make my code work with the timer, I was running into compiler errors.  It didn't recognize a lot of the signal stuff even though I had included signal.h, time.h and the _POSIX_C_SOURCE macro (like in timer.c).  Does anyone know why this wouldn't work in my code? Something to do with -std=c99 or memwatch??

The solution I found was to remove the macro and compile with -D_POSIX_C_SOURCE whe making my my .o file.  I found it here:  http://stackoverflow.com/questions/8846047/compile-errors-using-signal-h-in-linux

Hope this helps if you start running into the same errors!
