CMPUT 201: Practical Programming Methodology (Winter 2015)

Assignment 1b

Assignment 1, Part b

Due Friday, February 6, 2015, before or at 5 PM.

Contents of this page

  • Overview of Part b of Assignment 1
  • Input/Output and Behavior Specification
  • Types of Lines in Input

Overview of Part b of Assignment 1

Note that the Assignment Marking and Submission page applies to this assignment.

In Part b of this assignment, you develop a C99 C program named draw1b (that's the number one in 1b, not the letter l) which modifies and extends your program from Part a. As before, your program will read from an input file. However, instead of computing statistics on line types, your program will now implement storing, drawing and writing the Image data contained in the input file. New steps include:

    • Parse the content of some types of lines completely, not just the first word (see details below)
    • For lineTo commands within Image definitions, save the data in memory by storing the coordinates of the points.
    • Use popen() to open a pipe to a child process running Sketchpad.
    • At draw commands, transform the points in an Image into a series of line drawing commands for Sketchpad, then send these commands to Sketchpad over the pipe. Sketchpad will show the results in its drawing window.
    • For comments and print commands, write text as specified below to standard output (stdout).
    • Forward child commands to the child process through the pipe
    • Implement translate commands that change a Image stored in memory by shifting it.

Data Flow

  • The Input File contains Image definitions and all other commands
    • The input file is read by draw1b
  • draw1b is the executable you create
    • draw1b sends comments and the result of print commands to stdout
    • draw1b sends drawSegment and child commands through a pipe to Sketchpad
  • stdout is the standard C output. By default, it goes to your terminal window.
  • Java runs the Sketchpad program
  • Sketchpad opens a drawing window
    • Sketchpad draws lines and does other graphics
  • The drawing window in Sketchpad shows the result

Input/Output and Behavior Specification

Your program's name is draw1b now. As in Part a, your program takes a file name as a command line argument, and prints a startup message such as:

./draw1b started on Wed Jul 2 21:08:40 MST 2010
Input file: ./inputfile.1

inputfile.1 from Assignment 1a is an example of a valid input file.

Your program should no longer print statistics as in draw1a. Instead, draw1b should start up Sketchpad using popen to create a drawing window. In response to an input file, draw1b would instruct Sketchpad to draw a picture that looks similar to this (the specific lines drawn are different):

Your program will also produce output to stdout:

./draw1b started on Mon  6 Feb 2012 18:40:59 MST
Input file: inputfile.1

# ############################

# child end below is last line in input file

Your program will be tested on many different input files, including those used for Part a.

The following are the only types of lines that will appear in an input file.

Types of Lines in Input Files

Input files will be written in the "little language" used for Part a of this assignment. You need to handle the input as follows:

  1. A Comment (e.g., # This is a comment) is a line starting with a # . # is a token, so if there is more text in the line, it will be separated from the # character by whitespace. Comments should be echoed to stdout as shown in the above example. Nothing else is done with a comment.

  2. A Image definition spans two or more lines. Example:

    Image Q 10 10 =
    lineTo 50 0
    lineTo 0 75
    End Image Q
    Each Image starts with a line of the form Image image-name x y = and ends with the line End Image image-name. Each line between those two lines contains either a lineTo command or a comment. A lineTo command defines one in a sequence of connected line segments. When the image is later drawn, its commands can be thought of as controlling an imaginary pen. The initial pen position is set by x and y (e.g. 10, 10 for Image Q above).

    Each lineTo command is of the form lineTo delta_x delta_y (e.g., draw 100 100) and draws a line segment from the current pen position (x_curr, y_curr) to the new pen position (x_curr+delta_x, y_curr+delta_y). The endpoint of the line segment drawn then becomes the new current position. For example, Image Q above consists of two connected line segments, one from (10, 10) to (10+50, 10+0) = (60, 10) and the other from (60, 10) to (60, 85).

    Image     definitions need to be fully parsed and processed as follows: Store the subsequent points between which the line segments are later drawn. For example, for Image Q above, store three points defining the endpoints of the two line segments in the following order: 
    x  y
    == ==
    10 10
    60 10
    60 85

    In general, line segment i starts at point i and ends at point i+1, where 0 <= i < number-of-lines.

    Important: store coordinates of points as type double.

    Read the Guarantees section of the Assignment 1 Hints and Resources page for restrictions on our test input.

  3. The draw command is of the form draw image-name (e.g., draw Q). It draws the line segments defined by image image-name. draw Q should be implemented by generating one draw instruction per line segment of the Image. In the case of Image Q:

    drawSegment 10 10 60 10
    drawSegment 60 10 60 85
    These drawing instructions must then be sent to the stdin (standard input) of your program's child process which runs Sketchpad. This will cause the line segments to be drawn by Sketchpad. See the Hints section of the Assignment 1 Hints and Resources page for information on sending data to the child process.

    Since point coordinates are stored as doubles, you need to round each coordinate to a long using lround() before sending it to Sketchpad. Use %ld (that's an el, not a one) as format for numbers in fprintf to send drawSegment instructions to Sketchpad.

  4. The print command is of the form print image-name (e.g., print Q). It prints a list of the coordinates of the points defined by image image-name to stdout. For Image Q, the correct output of print Q is:

    Print Image Q =
    10 10
    60 10
    60 85
    End Image Q
    Use the formatting shown in this example and print the coordinates as type double using the %g format of printf. For empty images, which do not have any draw commands, do not print any points, not even the pen starting point.

  5. A translate command of the form translate image-name x y (e.g., translate A 20 30) should translate all points of Image image-name, which are stored in memory, by x units horizontally and y units vertically. No output is produced by this command, but it affects the output of future print and draw commands for this image. x and y are of type double.

  6. A child command is of the form child text (e.g., child pause 2). Copy text on this line (i.e., everything after child, including the whitespace between child and the first word) to the child process running Sketchpad. You do not need to parse the remaining characters after child into tokens.

Your output to stdout and to Sketchpad has to match the expected output when compared by the Linux command diff -b.

Make sure to read the Assignment 1 Hints and Resources page as well as the Assignment Marking & Submission page.
