Assignment 1 Hints & Resources

Assignment 1 - Hints and Resources

General Hints

  • Make use of the guarantees below to simplify the design of your code
  • You need to do some error checking
  • Handle all valid cases listed below.
  • Use our sample input files and Sketchpad -d for testing your code
  • Check out marking and submission information

Resources

Study the following resources as needed:

  • standard C library functions for files, strings
  • standard Unix/Linux functions popen, pclose

      Error Checking

      In Assignment 1, there are strong guarantees on the test input, so you do not need a lot of error checking and handling in this assignment. However this will change in Assignment 2, so you can begin to think about how to detect errors in the input.

      Note you must still detect and appropriately deal with return values from certain standard library C functions (as specified at the Marking and Submission Rules page).

      Guarantees about Test Input

      For both parts of Assignment 1, your code will only be tested on input which is valid according to the specifications. Furthermore, it will obey the following restrictions:

      • The input file's name will be no more than 128 characters.
      • Input lines will be no more than 256 characters (including the newline).
      • Each line starts with one of the 8 keywords. There will be no leading whitespace on a line.
      • Relevant for Part b only:
        • The name of a figure/image will be no more than 128 characters, and will consist solely of alphanumeric characters.
        • There will be no more than 16 figures/images.
        • A figure/image will never define more than 16 line segments, so will never have more than 16 lineTo commands.
        • All numbers not in comments can be stored as doubles without causing overflow. All operations requested by the input file will also not result in overflow.
        • All child commands will have correct Sketchpad instructions.
        • The last line of input is always child end.

      Explicitly Valid Cases (Relevant for Part b only)

      You are expected to handle the following non-obvious and/or non-trivial cases:

      • Figures, which are labelled as Image in the input, may be redefined.The new definition replaces the old one.
      • Some Figures/Images may not contain any lineTo command. For such empty figures/images, do not draw any line segments and do not print any points, not even the pen starting point.
      • One or more points in a figure/image can be outside the Sketchpad window.
      • A Figure/Image name can be any alphanumeric token (combination of characters and digits).

      Marking and Submission

      You are required to read the Assignments Marking and Submission Rules page and follow all instructions:

      • Having correct code is important, but good style and quality are also important.
      • You need to write a Makefile for your program that creates a draw1a or draw1b executable.

      Submit using the following command:

       Part 1a: % astep -c c201 -p asn1a submit.tar  

       Part 1b: % astep -c c201 -p asn1b submit.tar

      See submission rules for more information on submit.tar.

      Hints

      • To print the start-up message, use the following hints:
        • Use fflush right before you call system (see immediately below) to ensure the order of your output is correct. Use it to flush all open output streams. See its man page for more information.
        • Use the line
          system("date");
          to call the Linux date command, which prints out the current date and time. See the system man page for the header file to include.
      • To send drawSegment commands to Sketchpad, open a pipe to Sketchpad using popen (man 3 popen), send the commands using fprintf (man 3 fprintf. It is similar to printf.), and then close the pipe using pclose. popen and pclose should only be called once each.
      • popen and pclose are not part of the. To use them, you have to #include the appropriate header file (see their man pages).
      • For lround you must include the math library when linking (as well as including math.h in your .c file). Do this by including the gcc -lm(an el, not a one) option when linking. Here's such a sample command:
        gcc -std=c99 -Wall draw1.o -o draw1 -lm
        (See the gcc man page for more info. It's a bit tricky to find the -lm option, though, so ask a TA for help if needed and/or post to the Course Discussion Forum.)
      • As always, please ask TAs and other course staff if you need assistance.
