CMPUT 201: Practical Programming Methodology (Winter 2015)

Lab 1: Introduction to Linux and C

Lab Exercise 1: Linux and C Basics

This lab exercise contributes to your lab mark. Note that for demoed labs, you must demo your lab exercise in the lab section you are registered in.

Objectives

After completing this lab exercise, students should be able to
- use commands like cd and pwd to navigate the file system
- access the man page for a command
- search for text in a man page
- open and edit a text file in a text editor of your choice (emacs or vim strongly recommended)
- compile a C program to a named executable
- run a C program

Preparation

Review the Linux tutorials on eClass. Familiarize yourself with C resources such as the King textbook, and the links posted on the Online Resources page.

Lab Exercise

This lab exercise has three marked parts (Parts 1-3) and one optional part (Part 0).

Part 0: Create a CMPUT 201 directory structure (optional, but recommended)

The purpose of this exercise is to have you set up a directory structure to keep your CMPUT 201 files organized. Your directories can be different from what is suggested here, however organizing your files will be helpful in the long run.

One suggested way of organizing your files is to create a directory called cmput201 in your home directory. Inside the cmput201 directory, create directories for labs, assignments, and test programs. Inside the labs and assignments directories, create folders for each lab and assignment as you begin working on them.

For example, you can use the cd and mkdir commands as follows to create a structure as described above:

cd
mkdir cmput201
cd cmput201
mkdir labs
mkdir assignments
mkdir testprograms
cd labs
mkdir lab1

You can also use the pwd command to find out which directory you are currently working in. Use the man pages to find out more information about each of these commands.

Part 1: Using man pages and redirecting output

Find the ls command that would list the sizes and other information of files in your home directory, where the file sizes are in human-readable format (rather than just in bytes), and would work from any path (not just from inside your home directory).

Read Navigating the File System in the Linux tutorial for more information on your home directory and the term path.

(Hints: ls will need 2 options. Type man ls at the command prompt, and search for "human-readable". Also see Using man pages.)

Test your command to make sure it is working correctly.

Using vim or emacs, create a file called ls_com.txt, and save the ls command you found into ls_com.txt. The file should contain one line of the form
ls -? -? ?
where each question mark is replaced by exactly 1 character.

Redirect the output from the ls command you found above into a file called ls_out.txt (Hints: see the Linux tutorial for how to redirect output to a file.)

Part 2: Debugging and compiling C programs

Type the hello.c program from the whiteboard into a file called hello.c using emacs or vim. This program has a bug in it. Try to compile the program to see the output from the compiler, and fix the bug.

Compile hello.c to an executable called hello. You should be able to run the program using the command ./hello

Part 3: Using tar

You will be using tar to submit your files for Assignments and future Lab Exercises in thise course. To practice, create a tar file called submit.tar that contains the ls_com.txt, ls_out.txt and hello.c files you created above. Extract your tar file and ensure the contents is as expected.

Use the presentation materials on tar as a reference.

Demoing

When you have completed Parts 1, 2, and 3, demo your lab exercise to your TA. To receive a completion mark for this lab exercise, show the TA your ls_com.txt, ls_out.txt , and hello.c files, as well as the commands you used to create and extract your tar file.

There are no part marks for this lab exercise.

(Added 13/01/2015) Demo steps:

  • Show the contents of your ls_com.txt file
  • Generate the ls_out.txt file
  • Compile hello.c into an executable called hello
  • Run hello
  • Create tar file containing ls_com.txt, ls_out.txt , and hello.c
  • Create a test/ directory
  • Move tar file to test/ directory
  • Untar file
