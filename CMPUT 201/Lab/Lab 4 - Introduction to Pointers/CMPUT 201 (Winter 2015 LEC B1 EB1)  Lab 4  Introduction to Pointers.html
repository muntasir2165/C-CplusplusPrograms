CMPUT 201: Practical Programming Methodology (Winter 2015)

Lab 4: Introduction to Pointers

Exercise 4: Introduction to Pointers

Note: you have two labs time to work on this exercise. The lab presentations on the MEMWATCH and debugger/DDD will also span two labs.

This lab exercise contributes to your lab mark. Note that for demoed labs, you must demo your lab exercise in the lab section you are registered in.

For this lab exercise, you may demo each of Parts 1, 2, and 3 independently to receive feedback from a TA before moving on to the next part. Demo all three parts to receive your mark for this lab.

Recommendations: Demo each part before moving on to the next part. Aim to have Part 1 demoed and have most of Part 2 complete by the end of the first week's lab. 

Objectives

After completing this lab exercise, students will have increased their understanding of pointers, and on static vs dynamic memory allocation.

The presentations for the lab will also introduce students to using MEMWATCH and the debugger. 

Preparation (in advance of lab)

Review the "Basics of Using diff" tutorial. Download the resource files for Lab 3.

Lab Exercise

Create three programs, part1.c, part2.c and part3.c, which read one text input file named below, and print to stdout the input lines in reverse order (first print the last line, then the 2nd-last line, etc.)

The files must be compilable with gcc -std=c99 -Wall.

Use this Makefile, which contains instructions on how to compile each program. Do not create other .c files, nor any .h files, and do not use MEMWATCH.

Assume the input file exists in the current directory (you do not read it as a command line argument, nor from stdin; you can just hardcode it into your program), assume each line in the input file contains exactly one number which is no longer than 256 characters, and ignore checking for errors. (Although it's usually irrelevant, each number can be stored as a long.)

  1. For part1.c, read from input1.txt, and assume it contains no more than 100 lines. Use a static array to hold the numbers; do not use dynamic memory. (sample input1.txt)
  2. For part2.c, read from input2.txt. The first number will indicate the number of lines in the rest of the input file. The first number will be 1 or greater, and it must be stored as a long. Do not print that number. Make exactly one call to malloc to create a dynamic array.
    Reuse code from part1.c, if you wish. (sample input2.txt)
  3. For part3.c, read from input3.txt. The input file is like for part1.c, but it has an arbitrary number of lines.
    Create a dynamic array, as for part2.c, that can initially hold 10 numbers from the input file. When the array is about to overflow, double its size using realloc, then add the next number. For example, before adding the 11th number, double the array size to hold 20 numbers. Then before adding the 21st number, double the size again to hold 40 lines, and so on.

    Reuse code from part1.c or part2.c, if you wish. (sample input3.txt)

Remember to free all memory.

Note other input files that correspond to the descriptions above may be used to test your code; get your code to work for all such input files.

The objective of this lab is to practice working with pointers and dynamic memory, and so your lab exercise code will not:

  • read in command line arguments,
  • use temporary files,
  • write to files,
  • call Linux commands,
  • create other processes, or
  • use file input functions other than fopen, fclose and fgets.

Demoing

Once done, demo it to your TA. To receive a completion mark for this lab exercise, show your TA the following:

  1. Your code for part1.c
  2. The output of running your part1.c program
  3. Your code for part2.c
  4. The output of running your part2.c program
  5. Your code for part3.c
  6. The output of running your part3.c program
